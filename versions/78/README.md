# 78

## 概要

| 設定 | 経路                                                 | Q 更新法      | パラメータ                               | 商品判別   | 購入ルール                                                                         | 追加事項                                                                            |
| ---- | ---------------------------------------------------- | ------------- | ---------------------------------------- | ---------- | ---------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------- |
| 78   | ATM → 朝食、飲み物、おやつをそれぞれ 1 つずつ → レジ | 並列 Q 更新法 | ε = 0.01, Map1, similarity_threshold=0.9 | 写真で判別 | 最安値商品のみ購入。所持金不足なら購入しない。店内写真と買い物リストの写真は違う。 | 人数パラメータの違いによる結果を調査するためのプロジェクト。people=0, 1, 2, 4, 8, 16で比較。77からいくつか修正。 |

## 設定ファイルについて(./config.json)

- version: このプロジェクトの番号
- item_amount: ./input/shopping_list 内の写真の枚数（実行後に自動作成）
- epsilon: ε の値
- starting_x: 開始地点の x 座標
- starting_y: 開始地点の y 座標
- show_output:結果を都度ターミナルに出力するときは true
- similarity_threshold: 類似度のしきい値
- title: 実験の手法
- runs_per_set: 1 セットあたりの実行回数
- sets: セット数
- max_step: 最大歩数
- people: 店内の人数

## 前回からの変更点

- ロボットが一歩移動するごとに、配置した人を全員ランダムな方向に一歩移動させる。
- 人が一定時間以上居続ける場合、その場所の q 値を減少。 -> この機能を削除
- 棚の中に人が入らないようにする。
